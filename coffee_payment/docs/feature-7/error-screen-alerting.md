Необходимо обновить экран ошибки и привести его к презентабельному, современному виду.
Экран должен визуально сочетаться с уже существующими экранами проекта и соответствовать общим правилам UI/UX.

На экране ошибки должны отображаться следующие элементы:
 - Логотип клиента — изображение, заданное через Device.logo_url.
 - Информация об ошибке — пользовательский текст ошибки, получаемый из централизованного файла сообщений.
 - Дополнительный текст для клиента — поле Device.client_error_info.

Все тексты должны быть читабельными и визуально структурированными.

Требования к UI/UX
 - Экран должен быть разработан по принципу mobile first и быть полностью адаптивным.
 - Дизайн должен быть визуально согласован со стилем остальных экранов (цвета, отступы, шрифты, кнопки и т.д.).
 - Все визуальные элементы и логика отображения должны соответствовать правилам из CONSTITUTION.md, включая:
    - использование централизованного хранения пользовательских текстов ошибок;
    - отсутствие технических ошибок или стек-трейсов в UI;
    - использование существующего логгера в принятых стилях.

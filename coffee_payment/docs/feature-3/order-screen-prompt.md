Добавь экран информации о заказе пользователя, который отображается в сценариях Yookassa и TBank после успешного получения цены напитка.

1. Назначение

Экран предназначен для отображения сведений о заказе перед переходом к оплате.
Пользователь должен видеть ключевую информацию о заказе и иметь возможность начать процесс оплаты.

2. Условия отображения

Экран показывается только в сценариях:
 - Yookassa
 - TBank

Он появляется после успешного получения цены напитка (успешного ответа от бэкенда).

3. Содержимое экрана

На экране должна отображаться следующая информация:

 - Логотип клиента — изображение, заданное для Device через поле logo_url.
 - Адрес кофемашины (location).
 - Название напитка.
 - Размер напитка — один из вариантов: маленький, средний, большой.
 - Цена напитка.
 - Информация для клиента — текст, задаваемый для каждой кофемашины (Device.client_info).
 - Кнопка “Перейти к оплате” — запускает процесс создания платежа в соответствующей платёжной системе.

4. Поведение интерфейса

При нажатии на кнопку “Перейти к оплате”:
 - Выполняется создание платежа в платёжной системе, соответствующей текущему сценарию (Yookassa или TBank).
 - Пока запрос выполняется, интерфейс должен отображать состояние загрузки.
 - В случае ошибки должен отображаться пользовательский текст ошибки из централизованного файла сообщений.

5. UI/UX требования

 - Экран разрабатывается по принципу mobile first и должен быть адаптивным.
 - Должны быть реализованы состояния:
    - Загрузка данных
    - Успешное отображение информации
    - Ошибка загрузки или создания платежа
 - Визуальные элементы (текст, кнопки, состояния) должны соответствовать правилам из CONSTITUTION.md (включая централизованное хранение текстов ошибок и использование существующего логгера).

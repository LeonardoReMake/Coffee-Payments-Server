Я хочу добавить возможность конфигурации для каждой кофемашины (Device) соответствующий сценарий оплаты из заданых заранее (например, сценарий оплаты через Yookassa или ТБанк).

Перечень возможных сценариев задается в настройках проекта. Сейчас он должен содержать следующие сценарии: Yookassa, TBank, Custom. По дефолту выбирается Yookassa.

При выборе сценария Custom после прохождения валидаций и создания заказа необходимо редиректить пользователя на redirect_url заданный для кофемашины.

Для каждого клиента (Merchant) для его сценариев должны храниться его Credentials. Так как Credentials могут быть разные у разных сценариев должны хранить их как json. Пока храним в базе как есть для упрощения.

Например, для сценария Yookassa, Credentials состоят из "account_id" и "secret_key".

При выполнении сценария необходимо использовать Credentials соответствующего мерчанта.


Добавь возможность конфигурации сценария оплаты для каждой кофемашины (Device).

## Бизнес-задача

Для каждой кофемашины (Device) нужно уметь задавать, какой сценарий оплаты используется.
Сценарий определяет, как будет происходить процесс оплаты (например, через Yookassa или ТБанк). Для каждого платежа, отдельный сценарий начинается после прохождения первичных валидаций и создания заказа в статусе Created.

## Перечень сценариев

Перечень доступных сценариев задаётся в настройках проекта и на текущем этапе должен включать:
 - Yookassa
 - TBank
 - Custom

Сценарий по умолчанию — Yookassa.

## Поведение сценариев

Для сценариев Yookassa и TBank используется внутренняя логика оплаты через соответствующие API.

Для сценария Custom после валидации и создания заказа необходимо выполнить редирект пользователя на redirect_url, указанный для конкретной кофемашины (Device.redirect_url).

## Хранение Credentials

Для каждого клиента (Merchant) необходимо хранить Credentials для каждого сценария оплаты.

### Требования:

У одного мерчанта может быть несколько сценариев — для каждого должны храниться свои Credentials.

Credentials сохраняются в базе данных в формате JSON без дополнительной структуры (упрощённый вариант).

Пример структуры Credentials для сценария Yookassa:

{
  "account_id": "string",
  "secret_key": "string"
}

## Использование Credentials

При выполнении сценария оплаты необходимо использовать Credentials соответствующего мерчанта и выбранного сценария.

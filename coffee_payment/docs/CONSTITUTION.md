# Основной принцип

Всегда помнить, что мы создаём MVP — минимально жизнеспособный продукт.
Любое решение должно быть минимально достаточным, чтобы удовлетворить требованиям задачи и обеспечить стабильную работу продукта.

# Общие правила разработки

## Простота и минимализм.
Избегать избыточных абстракций, сложных архитектур и преждевременной оптимизации. Сначала — рабочее решение, затем — улучшения при необходимости.

## Использование сторонних библиотек.
Для типовых задач (логирование, работа с API, временем, тестирование и т.п.) использовать проверенные библиотеки, а не писать собственные реализации, если нет объективной причины.

## Организация тестов.
Все тестовые файлы должны находиться в отдельной директории: ```coffee_payment/tests/```
Каждый тест должен запускаться с таймаутом 30 секунд.
Перед запуском тестов в терминали проверять, что активирована среда conda:
```
conda info | grep active
```

Если нет активированной среды, то необходимо активировать ```cofpay``` среду:
```
conda activate cofpay
```

## Логирование.
Использовать уже существующий логгер и принятый стиль логирования в проекте.
В логах всех запросов к внешним API обязательно указывать все параметры запроса.

## Работа со временем.
Всегда явно указывать таймзону, в которой представлено время.
Использовать формат времени с указанием таймзоны (например: 2025-11-12T10:30:00+03:00).

## Кодовая дисциплина.

Поддерживать читаемость и единый стиль кода.
Давать осмысленные имена переменным, функциям и файлам.
Документировать нестандартные решения и предположения.

## UI/UX экранов пользователя.

Все веб-страницы должны разрабатываться по принципу mobile first и корректно отображаться на устройствах с разными разрешениями экрана
При выполнении любых сетевых операций (загрузка данных, отправка запросов и т.д.) интерфейс должен отображать соответствующие состояния:
 - Загрузка — визуально показывает, что запрос выполняется;
 - Успех — пользователь переходит к следующему шагу или получает подтверждение успешного выполнения действия;
 - Ошибка — отображается понятное сообщение пользователю.
Пользователь не должен видеть технические детали ошибок (стек-трейсы, коды, внутренние тексты исключений и т.п.).
Все текстовые сообщения об ошибках, видимые пользователю, должны храниться централизованно — в отдельном файле, предназначенном для пользовательских текстовок.

## Запуск python команд в терминале 
При выполнении python команд в терминале, необходимо сначала активировать conda среду:
```
conda activate cofpay
```

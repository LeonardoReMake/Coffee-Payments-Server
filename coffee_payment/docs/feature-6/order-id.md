Есть проблема в том, что сейчас модель Order ожидает в качестве идентификатора UUIDField. Но в реальности id заказа генерируется на кофемашине и может не быть в валидном формате UUID.

Необходимо, чтобы идентификатором Order был id, который был сгенерирован на кофемашине (то есть в url QR кода).

Пример url в QR коде: 
```
https://pay.tmetr.ru/v1/pay?deviceUuid=b80081ff-4e5f-4cf1-891c-bc3d23492d38&drinkNo=9b900a2e63042d350f45b6675ef26ced&size=1&random=waxoqk&ts=1742198482&salt=b6c2cca0340a82d0dc843243299800d7&drinkName=%D0%9C%D0%BE%D0%BB%D0%BE%D1%87%D0%BD%D0%B0%D1%8F%20%D0%BF%D0%B5%D0%BD%D0%B0&uuid=20250317110122659ba6d7-9ace-cndn
```

Здесь значение id заказа: ```20250317110122659ba6d7-9ace-cndn``` и не является валидным UUID
